Excel vs Python: Data Handling Experiment

Over the last six years, Excel has been my go-to tool in roles as diverse as management consulting and financial planning. I used it for a range of tasks from data analysis to note-taking. However, transitioning to data science, I discovered the speed and efficiency of Python for data handling, which led me to design this experiment.

The experiment files can be found in my GitHub repository: Excel vs. Python.

Experiment Overview:

The aim of the experiment is to compare Excel and Python's performance in a common task in the day-to-day routine of any analyst: loading datasets, merging and saving the results (Figure 1 - Step 2 Merge and Save). For that we will follow these steps:

  1. Generate Datasets
    1. Create a dataset with 60 columns and a variable number of rows.
    2. Split the dataset into two dataframes:
      • Dataframe 1 consists of the first 30 columns.
      • Dataframe 2 consists of the remaining 30 columns.
      Both dataframes should include a column with a unique identifier (a number that uniquely identifies each row).
    3. Shuffle the rows in Dataframe 2.
    4. Save both dataframes. Then repeat the process with a different number of rows.
  2. Merge and save (timed task for both methods)
      1. Open File (only relevant for Excel)
      2. Load Dataframes 1 and 2
      3. Merge
      4. Save as a .csv/.xlsx file
