Excel vs Python: Data Handling Experiment

In the last chapter of my career before starting my master's degree at Purdue University—a six-year period—Excel was my go-to tool. I used it for everything from data analysis to note-taking. However, as I transitioned into data science, I discovered how fast and efficient Python is for handling data. This discovery inspired me to design an experiment comparing Excel's Power Query, a tool for data manipulation, with Python's Pandas library. I hope this comparison will provide insights for those still undecided about incorporating Python into their data handling routines.

The experiment files can be found in my GitHub repository: Excel vs. Python.

Experiment Overview:

The aim of the experiment is to compare Excel and Python's performance in a common task in the day-to-day routine of any analyst: loading datasets, merging and saving the results (Figure 1 - Step 2 Merge and Save). For that we will follow these steps:

  1. Generate Datasets
    1. Create a dataset with 60 columns and a variable number of rows.
    2. Split the dataset into two dataframes:
      • Dataframe 1 consists of the first 30 columns.
      • Dataframe 2 consists of the remaining 30 columns.
      Both dataframes should include a column with a unique identifier (a number that uniquely identifies each row).
    3. Shuffle the rows in Dataframe 2.
    4. Save both dataframes. Then repeat the process with a different number of rows.
  2. Merge and save (timed task for both methods)
      1. Open File (only relevant for Excel)
      2. Load Dataframes 1 and 2
      3. Merge
      4. Save as a .csv/.xlsx file

    Takeaways:

    Starting with the lessons learned with this exercise:

    • Excel's Power Query is about 10 times slower than Python for the same task.
    • In Python, saving data to a .csv file can take up to 80% of the total time, so being mindful of when and how often you save data can greatly affect performance.
    • Contrary to intuition, loading and merging data takes less time than opening files and saving results for both methods.

    Results:

    Overall Comparison

    Our first step of the analysis is comparing the performance of both methods with the increase of number of rows. And after analyzing how many times excel is slower when compared to python for each number of rows.

    • Analyzing Figure 1, we can observe that both Excel and Python exhibit a linear behavior as the number of rows increases. However, Excel's curve is noticeably steeper, indicating a faster growth in response time.
    • Moving on to Figure 2, excluding the results for tasks with 1,000 rows, we can conclude that Excel's performance is consistently at least ten times slower than Python's in executing the given task."

    Breakdown

    To deepen our analysis, we further dissect the total time required to complete the task by examining the duration of each step and each method to gain more insights.

    • Figure 3 indicates that the time taken to open and save an Excel file may account for more than 40% of the total task duration, depending on the number of rows in the dataframes.
    • Figure 4 demonstrates that, on average, saving the results as a .csv file consumes more than 80% of the total processing time.
