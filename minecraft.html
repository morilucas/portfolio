About:

This project showcases the powerful combination of web scraping with Beautiful Soup, automated tasks with GitHub Actions, and webpage hosting with GitHub Pages. These techniques were instrumental in securing one of three seats in the Microsoft Minecraft research group, which is historically competitive due to high student demand.

On May 15, 2024, we received an email from The Data Mine staff (see Figure 1 below) informing us that projects with corporate partners would be listed on a rolling basis throughout the entire summer. We were advised to monitor the website for newly added projects (see Figure 2 below).

"We add projects all summer and students should be watching for new projects (â€¦) We will not be communicating open projects to students at this time as they are posting and filling quickly."


Once I was accepted into the Data Mine pool of students, I knew that project registration operated on a first-come, first-served basis. To stay ahead, I built a website to serve as my homepage throughout the summer. This allowed me to check for new project listings first thing every morning. Whenever a Microsoft project was posted, I would receive and see the daily updates on my website (see figure 3 below), enabling me to secure a slot. And that's exactly what happened.

Approach:

  1. Scrape the names of projects listed on the website using BeautifulSoup and compare the names to those stored in a CSV file (Figure 5 below) containing all previously scraped names. If a name is new, flag it as new.
    • Python script: link to file on GitHub
  2. Repeat this process daily at 00:00 UTC using a GitHub Actions workflow.
  3. Filter and load the flagged projects to the website with JavaScript and HTML (Figure 6 below).
    • Webpage code: link to file on GitHub

For each project, a new section is added to the end of the project listing webpage, with the project name stored in the "Notes" column. By inspecting the page code, we can see that project names can be selected by targeting rows tagged with the class "schdType1".


In the next step, we extract the content of the last column in each row, store it in a CSV file, and flag it if it is a new project. For example, in Figure 4, we would store the value "AstraZeneca (Application Advancement)" and mark "No" in the "New Entry" column since it is already on the list.



Finally, this information is displayed on the webpage as shown in Figure 3 above.
